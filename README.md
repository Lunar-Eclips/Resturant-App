# Resturant-App

Project plan
A simple app for a restaurant for queuing and billing their food and beverages.

Functions
- has the menu and ability to add foods in the future
- will have the ability to que things and take multiple orders and que them
- will use estimated time to automatically change ques
- will output after your estimated time has passed x
- will generate a number to track and que any order
- will have the ability to add VAT and discounts to bill
- Navigate able by just typing strings or numbers
- Inventory Tracking: Reduce stock when an item is ordered, alert when stock is low
- Analytics: Daily sales reports, popular dishes


How it is going to work
- you will be opening up with a scanner so type what you need.
- if want to bill. just type bill. if you want to see the menu or add in the menu just type menu.
- Now if you are in bill you can now search through the categories which are appetizer, main course, dessert, set menu and beverage using the scanner
- now the category will open just type the number of that food and quantity (arrays will be used for this)
- now a number will be generated for the bill
- VAT and tax will be added and we will be able to count how many taka to give back if you gave more than you bill
- depending on the estimated time a stopwatch will start and notify you when it is finished (cancelled)
